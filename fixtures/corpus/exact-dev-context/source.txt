真实开发上下文样本：视觉压缩工程平衡测试。当前实验目标是找到 compact stream + 大字号像素字体 的可靠工程点。默认模型 ID 为 google-antigravity/gemini-3.5-flash。OpenAI-compatible gateway 地址为 http://127.0.0.1:4000/v1。字体锁文件路径是 fonts/fonts.lock.json。字体下载脚本是 scripts/fetch-fonts.sh。安全候选为 zpix-12-t030-l0-w750，平衡候选为 zpix-14-t030-l0-w1024，保守候选为 zpix-16-t030-l0-w1024。旧的激进候选 zpix-9-t030-l0-w750 在短 probe 中通过，但在长 QA 中会把 l0 读成 10，把 w750 读成 v750，因此不能作为默认工程配置。需要修复的 Rust 文件是 packages/coding-agent/src/config/model-registry.ts。另一个参考文件是 packages/snapcompact/src/snapcompact.ts。示例命令为 OPENAI_API_KEY=sk-test-123 scripts/qa-smoke.sh。示例错误码是 E0277。示例版本号是 16.3.6-zen.2。示例提交为 31a141c。示例 sha256 为 ed39f02845e8c0b8cdba275432250fb03e8528826f058bc151753bd62b44b744。JSON 配置键包括 modelRoles.smol、auth.broker.url、transport、pi-native。研究结论：排版必须从位图里删除，因为标题、空行、缩进和段落间距都会浪费像素。省下来的面积应该还给字号，让模型更轻松读取中英、数字、符号和路径。工程可用标准不是最大压缩率，而是在 exact EM 接近 100%、semantic QA F1 接近 98%、hallucination false positive 为 0 的前提下，选择像素面积最小的方案。下一步先测试 zpix 12、14、16 在 width 750 和 1024 下的 exact value 可靠性，再决定默认工程候选。本文没有给出真实生产 API key，也没有证明 zpix 已经可以默认进入 OMP。
